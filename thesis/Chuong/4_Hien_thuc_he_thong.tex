\section{Hiện thực module xác thực}

Module xác thực hỗ trợ ba phương thức đăng nhập: email/mật khẩu, Google OAuth 2.0, và LinkedIn OAuth. Ngoài ra, hệ thống hỗ trợ phiên khách (guest session) cho phép người dùng sử dụng trình soạn thảo CV mà không cần đăng nhập.

\subsection{Xác thực bằng email và mật khẩu}

Quá trình đăng ký bao gồm: xác thực đầu vào (định dạng email, độ mạnh mật khẩu), băm mật khẩu bằng thuật toán bcrypt với salt factor 12, lưu thông tin người dùng vào bảng \texttt{users}, và tạo phiên đăng nhập. Đoạn mã sau minh họa quá trình xử lý đăng nhập:

\begin{lstlisting}[language=TypeScript, caption={Xử lý đăng nhập trong route handler}]
export async function POST(request: Request) {
  const { email, password } = await request.json();
  
  // Rate limiting check
  const rateLimitResult = checkRateLimit(clientIP);
  if (!rateLimitResult.allowed) {
    return Response.json(
      { error: 'Too many attempts' }, 
      { status: 429 }
    );
  }
  
  // Find user and verify password
  const user = await findUserByEmail(email);
  const isValid = await verifyPassword(
    password, user.password_hash
  );
  
  // Generate JWT and set session cookie
  const token = jwt.sign(
    { userId: user.id, email: user.email },
    process.env.JWT_SECRET,
    { expiresIn: '7d' }
  );
  
  const response = Response.json({ success: true });
  response.headers.set('Set-Cookie', 
    `user_session=${token}; HttpOnly; Secure; Path=/`
  );
  return response;
}
\end{lstlisting}

\subsection{Xác thực bằng OAuth}

Luồng OAuth 2.0 được hiện thực cho cả Google và LinkedIn. Quá trình bao gồm bốn bước: (i) chuyển hướng người dùng đến trang xác thực của nhà cung cấp với \texttt{client\_id}, \texttt{redirect\_uri} và \texttt{state} token chống CSRF; (ii) nhận authorization code từ callback; (iii) đổi code lấy access token; (iv) sử dụng access token để lấy thông tin người dùng và tạo hoặc liên kết tài khoản trong hệ thống.

Dịch vụ liên kết tài khoản (\texttt{AccountLinkingService}) xử lý trường hợp người dùng đã có tài khoản email nhưng đăng nhập bằng OAuth: hệ thống tự động liên kết hai tài khoản thay vì tạo tài khoản trùng lặp.

\subsection{Phiên khách và middleware}

Middleware của Next.js kiểm tra mỗi request đến và phân biệt ba loại truy cập: (i) tuyến đường công khai (trang chủ, đăng nhập, đăng ký) -- cho phép tất cả; (ii) tuyến đường yêu cầu xác thực (\texttt{/cv-workspace}, \texttt{/admin}) -- yêu cầu JWT hợp lệ; (iii) tuyến đường cho phép khách (\texttt{/cv-upload}, \texttt{/cv-guided-editing}) -- cho phép cả người dùng đã đăng nhập và khách.

\begin{lstlisting}[language=TypeScript, caption={Middleware phân quyền truy cập}]
export function middleware(request: NextRequest) {
  const token = request.cookies.get('user_session');
  const path = request.nextUrl.pathname;
  
  // Guest-allowed routes
  if (path.startsWith('/cv-upload') || 
      path.startsWith('/cv-guided-editing')) {
    return NextResponse.next();
  }
  
  // Auth-required routes
  if (path.startsWith('/cv-workspace') || 
      path.startsWith('/admin')) {
    if (!token) {
      return NextResponse.redirect('/login');
    }
  }
  
  return NextResponse.next();
}
\end{lstlisting}

\section{Hiện thực module tải lên và phân tích CV}

\subsection{Trích xuất văn bản}

Module trích xuất văn bản xử lý hai định dạng tệp: PDF và DOCX. Đối với PDF, hệ thống sử dụng thư viện \texttt{pdf-parse} kết hợp với \texttt{pdfjs-dist} để trích xuất toàn bộ nội dung văn bản. Đối với DOCX, thư viện \texttt{mammoth} được sử dụng để chuyển đổi nội dung thành văn bản thuần, loại bỏ định dạng nhưng giữ nguyên cấu trúc đoạn văn.

Tệp gốc sau khi trích xuất văn bản được tải lên Vercel Blob Storage thông qua API endpoint \texttt{/api/upload/cv-blob}, lưu trữ đường dẫn (URL) để người dùng có thể tải xuống sau này.

\subsection{Hybrid CV Parser}

Đây là thành phần cốt lõi của hệ thống, được hiện thực trong file \texttt{shared/services/hybridCVParserService.ts}. Parser hoạt động theo hai pha:

\textbf{Pha 1 -- Phân tích trực tiếp (Direct Parsing):} Lớp \texttt{SectionParsers} sử dụng regex và các heuristic để trích xuất thông tin liên hệ, tóm tắt nghề nghiệp, học vấn, và kỹ năng. Module \texttt{EnhancedTextProcessor} tiền xử lý văn bản, chia thành các section dựa trên từ khóa tiêu đề phổ biến (``Experience'', ``Education'', ``Skills'', ``Kinh nghiệm'', ``Học vấn'', ``Kỹ năng''...).

\begin{lstlisting}[language=TypeScript, caption={Trích xuất thông tin liên hệ bằng regex}]
static parseContact(cvText: string) {
  const contactText = EnhancedTextProcessor
    .extractSection(cvText, 'contact');
  
  const emailMatch = cvText.match(
    /[\w.+-]+@[\w-]+\.[\w.]+/
  );
  const phoneMatch = cvText.match(
    /(\+?\d{1,3}[-.\s]?)?\(?\d{2,4}\)?[-.\s]?\d{3,4}[-.\s]?\d{3,4}/
  );
  const linkedinMatch = cvText.match(
    /linkedin\.com\/in\/[\w-]+/i
  );
  
  return {
    full_name: extractName(contactText),
    email: emailMatch?.[0] || '',
    phone: phoneMatch?.[0] || '',
    linkedin: linkedinMatch?.[0] || '',
    address: extractAddress(contactText)
  };
}
\end{lstlisting}

\textbf{Pha 2 -- Phân tích bằng LLM (ChatGPT Parsing):} Phần kinh nghiệm làm việc được gửi đến OpenAI ChatGPT API với system prompt và structured output request. Mô hình \texttt{gpt-4o-mini} được chọn để cân bằng giữa chi phí và chất lượng.

\begin{lstlisting}[language=TypeScript, caption={Gọi ChatGPT API phân tích kinh nghiệm}]
const messages: ChatMessage[] = [
  {
    role: 'system',
    content: `You are a professional CV parser. 
    Extract work experience as structured JSON.
    Return: position, company, location, 
    start_date, end_date, bullets[]`
  },
  {
    role: 'user', 
    content: `Parse work experience:\n${cvText}`
  }
];

const response = await fetch(
  'https://api.openai.com/v1/chat/completions', {
  method: 'POST',
  headers: { 
    'Authorization': `Bearer ${apiKey}` 
  },
  body: JSON.stringify({
    model: 'gpt-4o-mini',
    messages,
    max_tokens: 2000,
    temperature: 0.1
  })
});
\end{lstlisting}

Kết quả từ hai pha được hợp nhất thành đối tượng \texttt{CVParsingResponse} với các trường: \texttt{contact}, \texttt{summary}, \texttt{work\_experience}, \texttt{education}, \texttt{skills}, và \texttt{possibility\_score}.

\subsection{Cơ chế dự phòng}

Nếu hybrid parser gặp lỗi (ví dụ: API OpenAI không khả dụng), hệ thống tự động chuyển sang \texttt{cvParserService.ts} -- service sử dụng hoàn toàn ChatGPT API cho toàn bộ quá trình phân tích. Cơ chế dự phòng này đảm bảo hệ thống luôn hoạt động ngay cả khi một thành phần gặp sự cố.

\section{Hiện thực trình soạn thảo CV}

Trình soạn thảo CV (CV Guided Editor) là thành phần giao diện phức tạp nhất của hệ thống, nằm tại đường dẫn \texttt{/cv-guided-editing/[cvId]}. Component chính \texttt{CVEditor} được tải động (dynamic import) để tối ưu bundle size.

\subsection{Section-based editing}

Mỗi phần của CV được hiện thực dưới dạng một component React riêng biệt: \texttt{ContactSection}, \texttt{SummarySection}, \texttt{WorkExperienceSection}, \texttt{EducationSection}, và \texttt{SkillsSection}. Người dùng cũng có thể thêm các section tùy chỉnh như Projects, Certifications, Languages, và Hobbies.

Dữ liệu CV được quản lý tập trung thông qua \texttt{CVWorkflowContext} sử dụng React Context API kết hợp với useReducer pattern. Mọi thay đổi từ các section component đều dispatch action đến reducer, đảm bảo trạng thái nhất quán và hỗ trợ undo/redo.

\subsection{Kéo thả với @dnd-kit}

Thư viện \texttt{@dnd-kit} được sử dụng để hiện thực chức năng kéo thả sắp xếp thứ tự các mục trong phần kinh nghiệm làm việc và các section tùy chỉnh. Khi người dùng kéo thả một mục, hệ thống cập nhật thứ tự trong state và phản ánh ngay lập tức trên bản xem trước.

\subsection{Tự động lưu}

Cơ chế auto-save sử dụng \texttt{useEffect} hook kết hợp debounce (2 giây) để lưu dữ liệu CV lên Supabase mỗi khi có thay đổi. Đối với phiên khách, dữ liệu được lưu vào \texttt{localStorage} của trình duyệt.

\subsection{Xem trước real-time}

Panel preview bên phải hiển thị CV trong định dạng A4 chuẩn in ấn, sử dụng template Dennis Schroeder với layout chuyên nghiệp. Preview cập nhật tức thì khi người dùng chỉnh sửa bất kỳ nội dung nào ở panel editor bên trái. Section đang được chỉnh sửa được highlight trên preview để người dùng dễ theo dõi.

\section{Hiện thực module xuất CV}

Hệ thống hỗ trợ xuất CV ra bốn định dạng khác nhau, mỗi định dạng phục vụ một nhu cầu cụ thể:

\textbf{Xuất PDF:} Sử dụng cơ chế print dialog của trình duyệt thông qua \texttt{window.print()} với CSS media query \texttt{@media print} được thiết kế riêng để đảm bảo bản in giống với preview trên màn hình.

\textbf{Xuất DOCX:} Tạo file DOCX bằng thư viện \texttt{docx}, chuyển đổi dữ liệu CV có cấu trúc thành các đoạn văn (Paragraph), bảng (Table), và các phần (Section) theo định dạng Microsoft Word. File được tạo phía client và tải xuống thông qua \texttt{file-saver}.

\textbf{Xuất LaTeX:} Sinh mã LaTeX sử dụng lớp \texttt{moderncv}, phù hợp cho môi trường học thuật. Mã LaTeX được tạo từ dữ liệu CV có cấu trúc, với các lệnh \texttt{\\cventry}, \texttt{\\cvitem} cho từng mục.

\textbf{Xuất TXT:} Xuất văn bản thuần có cấu trúc rõ ràng, phù hợp cho việc dán vào các form ứng tuyển trực tuyến.

\section{Hiện thực quản lý CV workspace}

Trang workspace (\texttt{/cv-workspace}) hiển thị danh sách tất cả CV của người dùng, lấy từ bảng \texttt{cv\_workflow} thông qua Supabase client. Các thao tác CRUD được hiện thực:

\begin{enumerate}[label=(\roman*)]
\item \textit{Tạo mới:} Người dùng có thể tạo CV mới từ template hoặc tải lên CV có sẵn. Hệ thống tạo bản ghi mới trong \texttt{cv\_workflow} với trạng thái ``draft''.
\item \textit{Đọc:} Danh sách CV được sắp xếp theo thời gian cập nhật gần nhất, hiển thị tên, trạng thái, và ngày tạo.
\item \textit{Cập nhật:} Khi mở CV trong editor, mọi thay đổi được auto-save. Người dùng cũng có thể đổi tên CV trực tiếp từ workspace.
\item \textit{Xóa:} Xóa CV với hộp thoại xác nhận, thực hiện xóa bản ghi từ \texttt{cv\_workflow} và file từ Vercel Blob (nếu có).
\end{enumerate}

\section{Bảo mật hệ thống}

Hệ thống áp dụng nhiều lớp bảo mật:

\textbf{Mã hóa mật khẩu:} Mật khẩu được băm bằng bcrypt với salt factor 12 trước khi lưu vào cơ sở dữ liệu. Quá trình xác thực so sánh mật khẩu đầu vào với hash đã lưu mà không bao giờ giải mã ngược.

\textbf{Row Level Security:} Tất cả sáu bảng trong cơ sở dữ liệu đều kích hoạt RLS. Chính sách RLS đảm bảo mỗi người dùng chỉ đọc và ghi được dữ liệu thuộc về mình, ngay cả khi có lỗi logic ở tầng ứng dụng.

\textbf{CSRF Protection:} Luồng OAuth sử dụng state token ngẫu nhiên được lưu trong session cookie. Khi nhận callback từ nhà cung cấp OAuth, hệ thống so sánh state token để đảm bảo request không bị giả mạo.

\textbf{Rate limiting:} API đăng nhập và đăng ký áp dụng giới hạn số lượng request từ cùng một IP trong khoảng thời gian nhất định, ngăn chặn tấn công brute-force.

\textbf{Xác thực đầu vào:} Tất cả dữ liệu từ người dùng được kiểm tra định dạng (email hợp lệ, độ mạnh mật khẩu, kích thước tệp tải lên) trước khi xử lý.

\textbf{Session management:} JWT token được lưu trong httpOnly cookie, ngăn chặn truy cập từ JavaScript phía client. Token có thời gian hết hạn (7 ngày) và được kiểm tra ở mỗi request thông qua middleware.
