Cách tính điểm phù hợp — TF-IDF vs Embedding

Phương pháp 1 — TF-IDF
1
Tách từ (Tokenize)
"React TypeScript 3 năm kinh nghiệm"
→ ["react", "typescript", "kinh", "nghiem"]
Bỏ stop words: "và", "của", "the", "a"...
2
Tính TF — tần suất từ trong văn bản
"react" xuất hiện 3 lần / 50 từ → TF = 0.06
"typescript" xuất hiện 2 lần / 50 từ → TF = 0.04
3
Tính IDF — độ hiếm trên 676 JD
IDF("react") = ln(677/501) + 1 = 1.30
IDF("aws") = ln(677/89) + 1 = 2.93
← "aws" hiếm hơn → quan trọng hơn
4
TF-IDF vector → Cosine Similarity
CV = [0.078, 0.117, 0, 0.293, ...]
JD = [0.065, 0.093, 0.146, 0, ...]
cos(CV, JD) = (CV·JD) / (|CV|×|JD|) = 0.61
5
× 100 → Điểm cuối
61
0.61 × 100 = 61 điểm
≥ ngưỡng 5 → ✓ Phù hợp
Phương pháp 2 — Embedding
1
Kiểm tra cache (SHA256)
SHA256("React TypeScript...") → "a3f9c2..."
Có trong cache? → Dùng ngay, không gọi API
Cache giúp tốc độ ~0ms sau lần đầu
2
Gọi OpenAI API → Vector 1536 chiều
POST /v1/embeddings
model: text-embedding-3-small
→ [0.023, -0.187, 0.341, ..., -0.052]
← 1536 số thực, biểu diễn "ý nghĩa"
3
Hai vector trong không gian 1536 chiều
CV_vec = [0.023, -0.187, 0.341, ...]
JD_vec = [0.019, -0.201, 0.318, ...]
← Gần nghĩa → tọa độ gần nhau
4
Cosine Similarity — đo góc giữa 2 vector
cos(CV_vec, JD_vec)
= (CV·JD) / (|CV| × |JD|) = 0.734
← Góc nhỏ = ý nghĩa gần nhau
5
× 100 → Điểm cuối
73
0.734 × 100 = 73.4 điểm
≥ ngưỡng 40 → ✓ Phù hợp
TF-IDF — Đếm từ trùng nhau
"Frontend" ≠ "UI Engineer" Bỏ sót
"ReactJS" ≠ "React" Bỏ sót
Tốc độ: < 1ms  ·  Chi phí: $0 Nhanh & miễn phí
Embedding — Hiểu ý nghĩa
"Frontend" ≈ "UI Engineer" Hiểu đúng
"ReactJS" ≈ "React" Hiểu đúng
ROC-AUC: 0.766 — cao nhất trong 3 phương pháp
